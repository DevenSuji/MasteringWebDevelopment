HTML Text Formatting Elements

<kbd>: Keyboard Input

The <kbd> element represents user input, typically keyboard input, but also potentially other input methods like voice commands.

Browsers usually render text within <kbd> tags in the system's default monospace font.

Example:

To copy text, select it and press Ctrl + C.

To save the file, press Ctrl + S.

You can also nest <kbd> elements to represent sequences or individual keys within a larger input:

Press Alt+F4 to close the window.

<p>Press <kbd>Ctrl</kbd> + <kbd>C</kbd> to copy.</p>
<p>Press <kbd><kbd>Alt</kbd>+<kbd>F4</kbd></kbd> to close.</p>

<samp>: Sample Output

The <samp> element represents sample or quoted output from a computer program or system.

Like <kbd>, browsers typically render this in a monospace font.

Example:

If you enter an invalid command, the system might respond with:

Command not found.

After running the script, the console showed:

Process finished with exit code 0

<p>The program outputted: <samp>File saved successfully.</samp></p>

<code>: Code Snippet

The <code> element represents a fragment of computer code. This can be an inline snippet or a larger block (often used within a <pre> element).

Browsers render this in a monospace font by default.

Example (Inline):

The HTML <span> element is an inline container.

In JavaScript, use the console.log() method for debugging.

Example (Block - inside <pre>):

See the <pre> section for how to display blocks of code while preserving formatting.

<p>Use the <code>getElementById()</code> method.</p>

<pre><code class="language-javascript">
function greet(name) {
  console.log(`Hello, ${name}!`);
}
</code></pre>

<var>: Variable

The <var> element represents a variable. This could be a variable in a mathematical expression or programming context, an identifier, a function parameter, etc.

Browsers typically render text within <var> tags in an italicized font.

Example:

The area of a rectangle is calculated as length × width.

In the function calculate(x, y), x and y are parameters.

Consider the equation: ax2 + bx + c = 0. 

<p>The variable <var>userCount</var> stores the number of users.</p>
<p>Equation: <var>E</var> = <var>m</var><var>c</var><sup>2</sup>.</p>

<pre>: Preformatted Text

The <pre> element defines preformatted text. Text within a <pre> element is displayed in a fixed-width font (usually monospace), and it preserves both spaces and line breaks exactly as they are written in the HTML source code.

This makes it ideal for displaying blocks of code, ASCII art, or any text where whitespace and line endings are significant.

It's common practice to wrap the code inside a <pre> element with a <code> element for semantic correctness (indicating the content *is* code).

Example (Code Block):

# Simple Python function
def add(a, b):
    # This comment has leading spaces
    result = a + b
    return result

print(add(5, 3))  # Output: 8

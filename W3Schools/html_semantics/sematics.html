HTML Semantic Elements

What are Semantic Elements?

Semantic elements in HTML are elements with inherent meaning. They clearly describe their purpose to both the browser and the developer, improving accessibility, SEO, and code readability.

Non-Semantic Examples

Elements like <div> and <span> are purely presentational containers. They tell you nothing about the content they hold.

Semantic Examples

Elements like <article>, <nav>, <header>, <table>, and <form> clearly define their content's role.

Using semantic elements appropriately is crucial for building well-structured, accessible, and maintainable web pages.

Common Semantic Elements

HTML5 introduced several semantic elements to define different parts of a web page, replacing generic <div> tags with meaningful alternatives:

  • <article>
  • <aside>
  • <details>
  • <figcaption>
  • <figure>
  • <footer>
  • <header>
  • <main>
  • <mark>
  • <nav>
  • <section>
  • <summary>
  • <time>

Instead of relying on <div id="header"> or <div class="nav">, we can now use <header> and <nav> directly.

<section>

The <section> element defines a thematic grouping of content within a document, typically (but not always) with its own heading.

Think of it as a chapter, an introduction, a set of news items, or contact information. It groups related content together.

Use <section> when the content represents a distinct part of the document's outline. Don't use it as a generic container; that's what <div> is for.

Example:

About WWF

The World Wide Fund for Nature (WWF) is an international organization working on issues regarding the conservation, research and restoration of the environment...

WWF's Panda Symbol

The Panda has become the symbol of WWF. The well-known panda logo originated from a panda named Chi Chi...

<section>
  <h2>Chapter 1: Introduction</h2>
  <p>This chapter introduces the main concepts...</p>
</section>

<section>
  <h2>Chapter 2: Core Ideas</h2>
  <p>Building upon the introduction...</p>
</section>

<article>

The <article> element represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable (e.g., in syndication).

Think of forum posts, blog entries, newspaper articles, user comments, or interactive widgets. If you could syndicate it as a standalone piece, it's likely an <article>.

An <article> should typically have its own heading.

Example:

Blog Post Title

Published on April 3, 2025

This is the content of the blog post, discussing various interesting topics...

Posted by John Doe

User Comment

I found this article very helpful! Thanks for sharing.

Comment by Jane Smith

<article class="blog-post">
  <h2>My Journey into Web Development</h2>
  <p>It all started with a simple HTML page...</p>
  <!-- More content -->
  <footer>
    <p>Published: <time datetime="2025-01-15">Jan 15, 2025</time></p>
  </footer>
</article>

Nesting <article> & <section>

You can nest <article> and <section> elements. The key is understanding their semantic difference:

  • <article>: Standalone, complete piece of content.
  • <section>: Thematic grouping within a larger context.

Example 1: Sections within an Article

A single blog post (article) divided into logical parts (sections).

<article>
  <h2>Comprehensive Guide to CSS Grid</h2>
  <section>
    <h3>Introduction to Grid Concepts</h3>
    <p>...</p>
  </section>
  <section>
    <h3>Grid Container Properties</h3>
    <p>...</p>
  </section>
</article>

Example 2: Articles within a Section

A page section listing multiple, independent blog summaries (articles).

<section>
  <h2>Latest Blog Posts</h2>
  <article>
    <h3>Post Title 1</h3>
    <p>Summary of post 1...</p>
  </article>
  <article>
    <h3>Post Title 2</h3>
    <p>Summary of post 2...</p>
  </article>
</section>

<header>

The <header> element represents introductory content, typically a group of introductory or navigational aids.

It may contain some heading elements (<h1> - <h6>) but could also contain a logo, search form, author name, etc.

You can use <header> at the top of the entire page, or as the header for a specific <article> or <section>.

Example (Page Header):

My Awesome Website

Your tagline here!

{/* A nav element often goes inside a page header */}

Example (Article Header):

Article Title

By Author Name

Article content starts here...

<!-- Page Header -->
<header>
  <img src="logo.png" alt="Logo">
  <h1>Website Name</h1>
  <nav>...</nav>
</header>

<!-- Article Header -->
<article>
  <header>
    <h2>Post Title</h2>
    <p>Published: <time datetime="2025-04-03">April 3, 2025</time></p>
  </header>
  <p>Article content...</p>
</article>

<footer>

The <footer> element represents a footer for its nearest sectioning content (like <section>, <article>, <nav>, <aside>) or sectioning root (like <body>).

A footer typically contains information about its section such as the author, copyright data, or links to related documents.

Contact information within a <footer> should be wrapped in an <address> element.

Example (Page Footer):

© 2025 My Website. All rights reserved.

Contact: info@example.com

Example (Article Footer):

Article Title

Article content...

Tags: html, semantics

<!-- Page Footer -->
<footer>
  <p>Copyright © 2025</p>
  <p><a href="/about">About Us</a> | <a href="/contact">Contact</a></p>
</footer>

<!-- Article Footer -->
<article>
  ... content ...
  <footer>
    <p>Author: John Doe</p>
    <p>Related links: <a href="...">Link 1</a></p>
  </footer>
</article>

<nav>

The <nav> element represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents.

Common examples include menus, tables of contents, and indexes. Not all groups of links need to be in a <nav> element â€“ only major navigation blocks.

Example:

  • Home
  • About
  • Services
  • Contact
<nav aria-label="Primary navigation">
  <ul>
    <li><a href="/home">Home</a></li>
    <li><a href="/products">Products</a></li>
    <li><a href="/blog">Blog</a></li>
  </ul>
</nav>

Using aria-label helps assistive technologies identify the purpose of different navigation blocks if you have more than one.

<aside>

The <aside> element represents a portion of a document whose content is only indirectly related to the document's main content.

Asides are frequently presented as sidebars or call-out boxes. They might contain related links, author biography, advertisements, or profile information.

Example:

Main Content Area

This is the primary content of the page or article...

Related Links
  • Link 1
  • Link 2
Advert

Check out this amazing product!

<article>
  <h2>Article Title</h2>
  <p>Main article content...</p>
</article>

<aside>
  <h3>About the Author</h3>
  <p>John Doe is a web developer...</p>
  <h3>Related Posts</h3>
  <ul>...</ul>
</aside>

<figure> & <figcaption>

The <figure> element represents self-contained content, potentially with an optional caption, which is specified using the <figcaption> element.

The figure, its caption, and its contents are referenced as a single unit from the main flow of the document. It's often used for images, illustrations, diagrams, code snippets, or quotes that are referenced in the main text.

The <figcaption> is optional and can appear before or after the content within the <figure>.

Example:

<figure>
  <img src="image.jpg" alt="Description of image">
  <figcaption>Figure 1: A descriptive caption for the image.</figcaption>
</figure>

<figure>
  <pre><code>
  // Code example as figure content
  console.log('Hello');
  </code></pre>
  <figcaption>Listing 1: JavaScript hello world example.</figcaption>
</figure>

<details> & <summary>

The <details> element creates a disclosure widget in which information is visible only when the widget is toggled into an "open" state.

A <summary> element must be provided as the first child, which serves as the summary or label for the rest of the <details> element's contents. Clicking the summary toggles the visibility of the other content.

Example:

Click to see details

This is the hidden content that appears when you click the summary above. It can contain paragraphs, lists, or any other HTML content.

{/* Starts open */} More Information (Starts Open)

You can use the open attribute to make the details visible by default.

<details>
  <summary>Copyright Information</summary>
  <p>© 2025 Example Corp. All rights reserved.</p>
  <p>Contact us for licensing details.</p>
</details>

<!-- Starts open -->
<details open>
  <summary>Release Notes</summary>
  <ul>
    <li>Version 1.1: Fixed bug X.</li>
    <li>Version 1.0: Initial release.</li>
  </ul>
</details>

<main>

The <main> element represents the dominant content of the <body> of a document. The main content area consists of content that is directly related to or expands upon the central topic of a document, or the central functionality of an application.

Content within <main> should be unique to the document, excluding content that is repeated across documents such as site logos, navigation links, copyright information, sidebars, etc. (these belong in <header>, <footer>, <nav>, <aside>).

A document must not have more than one <main> element.

Conceptual Structure:

<header> (Site Header, Nav) </header>
<main> (Unique Page Content Here) </main>
<footer> (Copyright, Site Links) </footer>
<body>
  <header>
    <!-- Logo, site title, main navigation -->
  </header>

  <main>
    <h1>Page Title</h1>
    <article>
      <!-- Main content specific to this page -->
    </article>
    <aside>
      <!-- Sidebar related to main content (if not site-wide) -->
    </aside>
  </main>

  <footer>
    <!-- Copyright, contact info, secondary links -->
  </footer>
</body>

<mark>

The <mark> element represents text which is marked or highlighted for reference or notation purposes, due to its relevance or importance in the particular context.

Think of highlighting text in a quote to draw attention to it, or highlighting search terms in results. Browsers typically render <mark> with a yellow background.

Example:

Search results for "semantic HTML":

The document discusses various semantic HTML elements like <article> and <nav>. Using semantic HTML improves accessibility.

<p>Do not forget to buy <mark>milk</mark> today.</p>

<blockquote>
  He said, <q>It was the <mark>best of times</mark>, it was the <mark>worst of times</mark>...</q>
</blockquote>

<time>

The <time> element represents a specific period in time or a precise date in the Gregorian calendar.

It can include the optional datetime attribute to translate the date or time into a machine-readable format, allowing for better processing by user agents (e.g., for calendar integration or search engine indexing).

Example:

The event will take place on October 26th.

We open daily from 9 AM to 5 PM.

This article was published earlier today.

{/* Z indicates UTC */} 
<!-- Simple Date -->
<p>Posted on <time datetime="2025-04-03">April 3, 2025</time>.</p>

<!-- Date and Time -->
<p>Meeting scheduled for <time datetime="2025-05-10T14:30:00">May 10th at 2:30 PM</time>.</p>

<!-- Time only -->
<p>The train departs at <time datetime="18:00">6 PM</time>.</p>

<!-- Duration (less common, format varies) -->
<p>The video duration is <time datetime="PT2M30S">2 minutes 30 seconds</time>.</p>

The format for the datetime attribute should follow standard date and time string formats (e.g., YYYY-MM-DD, YYYY-MM-DDTHH:MM:SSZ).
